# Changelog

## [1.2.0] ‚Äî 2026-06-22

### Breaking changes (generated output)

- **Go version from `go.mod`** ‚Äî `setup-go` steps now use `go-version-file: go.mod` instead of a pinned `GO_VERSION` env var. This eliminates the two-source drift that caused `GOTOOLCHAIN=local` failures. The `golang-version` Renovate customManager is dropped (4 remain). **If you generated config with v1.1.0 or earlier, regenerate.**

### Fixes

- **govulncheck v1.4.0 guard** ‚Äî Added inline warning comment and `allowedVersions: "<1.4.0 || >1.4.0"` in `renovate.json` to prevent automerge of v1.4.0, which panics on generics code via `x/tools@v0.46.0`.

### Features

- **CI concurrency** ‚Äî `ci.yml` now cancels in-progress runs when a newer commit supersedes them. Release workflows are intentionally excluded.
- **Version stamp** ‚Äî Generated files include `# generated by go-github-pipelines v<version>` to enable version-aware audits.
